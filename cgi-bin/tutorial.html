Gene2Path: a software to find orthologous genes and to generate KEGG pathways in uncommon species.



Scheme of the pathway analysis program and its search into different databases such as NCBI and KEGG. The diagram of Gene2Path is represented in the yellow cells, showing the automated steps of the program.

The cells in green represent the databases or external programs used by Gene2Path. Arrows indicate the databases needed to follow up each one of the program steps. Into the rose cells are mentioned the archives extensions used by Gene2Path: txt are the gProccesedSignal of each of the genes from the microarray; the xml archives were obtained from the external program BLAST2GO and are used by the Gene2Path program to process and organize the blastX information. The Dictionary section provide, if needed, the gene identification codes (ID), obtained from the NCBI GeneBank for each one of the genes.

The scripts and programme are available at the “GENE2PATH” web site and they include the following features: (1) The addition of synonyms for gene identification by searching the NCBI database; (2) the analysis of the intensity
of the gProccesedSignal reading of the synonym names microarray input files (extension txt); (3) Automated gene homology searches through the Homologene, Gene and Unigene NCBI databases; (4) An automatic filter was developed to find sequences similar by BLAST and to visualize the xml output file and to deal with several files at the same time; (5) Search for pathway information in the KEGG database using gene homologues (Gene ID).

GENE2PATH WEB TUTORIAL

1. Dictionary, gene symbol and download of the FASTA datafile.

The programme can convert the accession numbers into other code types available in the NCBI database (such as gene identification -GI- in GenBank). Moreover, it is possible to obtain the sequences from FASTA and to visualize the gene symbol for each of the genes. The IDs given by the user (input data) are run through the Nucleotide or EST sequence database, named “nuccore” or “nucest”, respectively.

The web provides an easy way to visualize the data and in our example, the FASTA download was selected and the screening displayed.






2. Normalization and the level of gene expression from DNA microarray data

The programme automates the analysis of the data generated by the one-channel microarray scanner. Values from the “g” signal process file in the “.txt” format were used. Among the programme options, the user can choose which of the data files correspond to control or experimental groups, and the gene expression fold with their t-test paired p-value. In our example, values ≥ 2 fold were selected.

 



3. Homology:

3.1. To identify homology through NCBI

The procedure needs an accession number as input. It first searchs for the corresponding gene IDs in the GenBank by using Unigen, Gen and HomoloGene to detect homology between the annotated genes or sequences of used specie with entire eukaryote genomes. The results are visualized on the web and the percentage of homology is annotated.







3.2. Identifying homology through BLAST2GO

The Gene2Path filters and organizes the results from the Blast2GO program (“.xml” files) with respect to the homologous species. As FASTA sequences from each of the genes are needed to run the Blast2GO programme, we can use data obtained in the Dictionary section.

Once the BLAST2GO programme (http://www.blast2go.com/b2ghome) has been installed in the computer and configured, it is necessary:

  1. To create a project loading the FASTA files of the genes of interest.

  2. To select the BLAST option in the window that automatically appears with several modifiable parameters. In our example, we selected Blastx to search for homologous proteins from a protein database starting with the nucleotide sequences. This is a very useful tool when the ORF is not known and it is commonly used in EST sequence analysis due to its high sensitivity.

  3. To determine how many results are visualized without changing the remaining parameters. We selected 100 results instead of the 20 that are given by default.

When the Blast2go finish the running of “annotation” application, the user can upload the resulting xml files. Those are then used for the homology step (named “homology”) in our GENE2PATH software.
